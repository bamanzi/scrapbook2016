7 Snapping a window to the top
8 ----------------------------
9
10 You can now choose whether snapping a window to the top edge of the screen
11 maximizes it to the top half (the previous and default behavior) or to the
12 full screen by setting "SnapToTopMaximizesFullscreen" to "NO" or "YES",
13 respectively.  This setting can also be changed by unchecking or checking
14 "Snapping a window to the top maximizes it to the full screen" in the
15 "Expert User Preferences" tab in WPrefs.app.

20 Window snapping
21 ---------------
22
23 You can now "snap" a window, i.e., maximize it to a side or corner of the
24 screen, by dragging it to that side or corner.  It is enabled by setting
25 "WindowSnapping = YES" in ~/GNUstep/Defaults/WindowMaker or selecting "Maximize
26 (snap) a window to edge or corner by dragging." under "Expert User Preferences"
27 in WPrefs.app.
28
29 Note that if "Switch workspaces while dragging windows" is selected under
30 "Workspace Preferences" in WPrefs.app, or if "DontLinkWorkspaces = NO" in
31 ~/GNUstep/Defaults/WindowMaker, then you may only snap a window to the top or
32 bottom of the screen.
33
34 You may set the distance (in pixels) from the edge or corner of the screen at
35 which a window will begin snapping using "SnapEdgeDetect" and "SnapCornerDetect"
36 in ~/GNUstep/Defaults/WindowMaker or setting "Distance from edge/corner to begin
37 window snap." under "Expert User Preferences" in WPrefs.app.  (The defaults are
38 1 pixel and 10 pixels, respectively).

41 Dragging maximized windows
42 --------------------------
43 You can now control the behavior when a maximized window is dragged by setting
44 the "DragMaximizedWindow" option in ~/GNUstep/Defaults/WindowMaker or by
45 selecting an option from the "When dragging a maximized window..." pop-up button
46 under "Window Handling Preferences" in WPrefs.app.
47
48 There are four choices:
49 * "Move" ("...change position (normal behavior)" in WPrefs.app) is the default
50   and traditional behavior.  A maximized window is moved when dragged and
51   remains maximized, i.e., it keeps its maximized geometry and can later be
52   unmaximized.
53 * "RestoreGeometry" ("...restore unmaximized geometry") is the behavior standard
54   in desktop environments like GNOME, Cinnamon, and Unity.  A maximized window
55   is moved when dragged and is completely unmaximized, i.e., its unmaximized
56   geometry is restored.
57 * "Unmaximize" ("...consider the window unmaximized") causes a maximized window
58   to be moved when dragged and remains partially maximized, i.e., it keeps its
59   maximized geometry, but is consider to be unmaximized.  In particular, it can
60   be immediately re-maximized.
61 * "NoMove" ("...do not move the window") prevents a maximized window from being
62   moved when dragged.
63
64 Note that, to accomodate this option in the "Window Handling Preferences" tab in
65 WPrefs.app, the option to "Open dialogs in the same workspace as their owners"
66 (which sets the "OpenTransientOnOwnerWorkspace" option from
67 ~/GNUstep/Defaults/WindowMaker) has been moved to "Expert User Preferences".

70 Mini-Previews instead of Apercus
71 --------------------------------
72
73 Since the original name was not really clear because it is a French word that
74 is rarely used by British people, it was decided to change it to the more usual
75 Mini-Preview name. The setting is configurable with WPrefs in the Icon
76 Preferences tab, the size is now expressed in pixels directly.

79 Ignore Decoration Hints from GNOME applications
80 -----------------------------------------------
81
82 The GNOME applications ask Window Maker to get no title bar and no resize bar to
83 their windows by using "Hints". You can re-add them using the Attribute dialog
84 in the Window menu, but if you are using many GNOME applications you may want to
85 tell Window Maker to just ignore them. This is done with the new setting called
86 "IgnoreGtkHints", which is available in the "Expert" panel in WPrefs.

89 Cooperative Window Manager Replacement
90 --------------------------------------
91
92 The ICCCM defines a protocol for window managers to ask to replace the currently
93 running one; Window Maker now supports it. You can ask Window Maker to take the
94 place of current one by running "wmaker --replace", or any other window manager
95 can ask Window Maker to leave the place for them when started the same way.
96 Please note that this feature must be explicitely enabled at compile time because
97 by default it is not compiled in ("configure --enable-wmreplace").

110 Mini-window apercu
111 ------------------
112
113 A small preview of window contents can be enabled from WPrefs,
114 in Miscellaneous Ergonomic Preferences, check miniwindow apercus.
115 Apercu size can be configured using the ApercuSize variable with
116
117 $ wdwrite WindowMaker ApercuSize 4
118
119 in multiples of the icon size (in this case the apercu size will be four
120 times the icon size).
121
122 The default size is 2 (twice the icon size).
