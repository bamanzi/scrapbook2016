如何用Windows远程桌面连接树莓派Raspbian Linux系统

2016-8-6 16:01 137 Reads 0 Comments

因为Windows 10 IoT Core还不能完全替代Raspbian，所以在有些场合我们软狗只能苟且偷生用Linux。Linux的远程操作通常通过SSH命令行完成，如果需要Windows那样的远程桌面的话，有2种解决办法：tightvnc或者xrdp。我以前一直用tightvnc，使用没有什么卵问题，唯一的问题就是要我在PC上装一个tightvnc客户端才能连接，拉低了软狗的尊严。而XRDP可以直接用Windows自带的MSTSC 远程桌面连接工具连接，假装有尊严。所以我今天就试了下XRDP，非常简单。

首先，SSH到树莓派。运行命令：

sudo apt-get -y install xrdp

完成后，还有个逼要装，就是默认情况下，你只能用IP地址来访问树莓派，而Windows的Ping命令，ping一个主机名是可以得到IP地址的，为了能ping树莓派装成功这个逼，还需要安装一个包：

sudo apt-get -y install samba

这一步耗时比较长

完成以后，就能从你的Windows PC Ping树莓派了：

现在在Windows PC上运行mstsc，就可以通过主机名或IP地址连接树莓派。

登录用的是你的Linux账号

曲线装逼完成。

等哪天Windows 10 IoT Core完善了蓝牙和PiCam支持，就再也不用苟且偷生用Raspbian了....
